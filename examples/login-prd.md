# 用户登录系统 产品需求文档

## 概述

提供一个安全、便捷的用户登录系统，支持多种登录方式，满足不同用户场景的需求。

## 功能需求

### 1. 账号密码登录
- 用户输入手机号/邮箱和密码
- 支持"记住密码"功能
- 密码错误超过5次锁定账号

### 2. 短信验证码登录
- 用户输入手机号
- 系统发送6位数字验证码
- 验证码有效期5分钟
- 每个手机号每天最多发送10次

### 3. 第三方登录
- 微信登录
- Google 登录
- GitHub 登录

### 4. 忘记密码
- 通过手机号重置
- 通过邮箱重置
- 需要验证身份

## 用户流程

### 登录流程
1. 用户进入登录页面
2. 选择登录方式（账号密码/短信验证码/第三方）
3. 根据选择输入相应信息
4. 点击登录按钮
5. 系统验证
6. 登录成功跳转首页，失败提示错误信息

### 注册流程
1. 用户进入注册页面
2. 选择注册方式
3. 填写基本信息
4. 获取并输入验证码
5. 设置密码
6. 完成注册，自动登录

## 数据模型

### 用户表 (users)
- id: 主键
- username: 用户名（唯一）
- email: 邮箱（唯一，可为空）
- phone: 手机号（唯一，可为空）
- password_hash: 加密后的密码
- avatar: 头像URL
- status: 账号状态（正常/锁定/禁用）
- created_at: 创建时间
- updated_at: 更新时间

### 登录日志表 (login_logs)
- id: 主键
- user_id: 用户ID
- login_type: 登录方式
- ip_address: IP地址
- user_agent: 浏览器信息
- created_at: 登录时间

## 接口设计

### POST /api/auth/login
账号密码登录

### POST /api/auth/sms/send
发送短信验证码

### POST /api/auth/sms/login
短信验证码登录

### POST /api/auth/oauth/{provider}
第三方登录回调

### POST /api/auth/password/reset
重置密码

## 安全要求

1. 密码必须加密存储（bcrypt）
2. 登录接口需要限流
3. 敏感操作需要日志记录
4. Token 有效期设置为7天
5. 强制要求密码复杂度（8位以上，含数字+字母）

## 非功能需求

- 性能：登录接口响应时间 < 200ms
- 可用性：99.9% 
- 安全：符合等保三级要求
