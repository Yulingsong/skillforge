# SkillForge 产品规范

## 1. 产品概述

### 产品定位
**SkillForge** - AI驱动的自动化技能生成平台

### 核心价值
将 PRD、TRD、源码、API文档等输入自动转换为可执行的 AI Skill，让用户通过自然语言引导完成复杂操作。

### 目标用户
- **产品经理**：快速生成功能操作指引
- **开发者**：为新功能一键生成配套 Skill
- **终端用户**：通过 Skill 引导完成专业操作

---

## 2. 产品功能

### 2.1 输入类型

| 输入类型 | 格式 | 说明 |
|---------|------|------|
| PRD | Markdown | 产品需求文档 |
| TRD | Markdown | 技术需求文档 |
| 源码 | .ts/.tsx/.js/.vue | 组件/API代码 |
| API | OpenAPI/Swagger | API 定义文档 |
| 截图/视频 | 图片/视频 | 操作演示 |

### 2.2 输出产物

| 输出 | 格式 | 说明 |
|------|------|------|
| SKILL.md | Markdown | 技能定义文档 |
| tools.json | JSON | 可用工具配置 |
| references/ | 目录 | 参考文档 |
| testcases/ | 目录 | 测试用例 |
| package.json | JSON | Skill 包配置 |

### 2.3 核心功能

1. **智能解析**
   - LLM 提取文档结构
   - AST 解析源码
   - OpenAPI 解析 API

2. **流程生成**
   - 用户操作步骤
   - 分支/条件处理
   - 异常处理流程

3. **Skill 生成**
   - SKILL.md 模板填充
   - 工具权限配置
   - 示例生成

4. **验证测试**
   - 语法校验
   - 逻辑测试
   - 引导流程测试

## 4. 技术方案

### 4.1 核心模块设计

```typescript
// 解析器接口
interface IParser {
  parse(input: string): Promise<ParsedResult>;
}

// 生成器接口
interface IGenerator {
  generate(parsed: ParsedResult, config: GenerationConfig): Promise<Skill>;
}

// 验证器接口
interface IValidator {
  validate(skill: Skill): Promise<ValidationResult>;
}
```

### 4.2 输入解析策略

| 输入类型 | 解析方式 | 工具 |
|---------|---------|------|
| Markdown | LLM 结构化提取 | OpenAI/Anthropic |
| TypeScript | AST 解析 | @babel/typescript |
| Vue | AST 解析 | @vue/compiler-sfc |
| OpenAPI | Schema 解析 | @apidevtools/swagger-parser |

### 4.3 Skill 输出格式

```yaml
# SKILL.md
name: 功能名称
description: 功能描述
triggers:
  - "关键词1"
  - "关键词2"

steps:
  - id: step1
    title: 步骤标题
    description: 步骤描述
    tools:
      - tool1
      - tool2
    validation: 验证条件
    errorHandling: 错误处理

examples:
  - query: 用户输入示例
    response: 期望响应
```
